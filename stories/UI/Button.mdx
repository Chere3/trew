# Button

The Button component provides tactile, retro-realistic button styling with physical pressable appearance. Features gradients, shadows, and clear interaction states.

## When to Use

- Primary actions and form submissions
- Navigation and important user actions
- Confirmation dialogs and destructive actions
- Secondary actions and outline buttons

## When Not to Use

- For simple text links (use Link variant)
- For icon-only actions without labels (use IconButton)
- For navigation within the app (use navigation components)

## Accessibility

- Keyboard accessible (Enter/Space)
- Focus visible with ring
- ARIA attributes for screen readers
- Disabled state properly announced

## Design Tokens

- Primary color: Red (`0 72% 55%` HSL)
- Shadow: `button-raised`, `button-hover`, `button-pressed`
- Border radius: `rounded-full`
- Typography: Medium weight, tight tracking
